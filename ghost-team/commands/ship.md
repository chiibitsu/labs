The founder wants to ship: $ARGUMENTS

Drive this end to end by orchestrating the ghost team via `chief-of-staff`:

1. **Frame** — define the smallest version that delivers real value, and the one
   assumption that most needs to be true.
2. **Validate (if cheap and warranted)** — pull in `customer-researcher` only if
   a quick check would change what gets built.
3. **Design** — have `designer` define the flow and the minimal UI for it.
4. **Build** — have `product-engineer` implement the vertical slice and leave it
   runnable, with exact run/deploy steps.
5. **Position & launch** — have `growth-marketer` write the actual launch copy and
   name the one channel + metric to watch.
6. **Money/legal check** — loop in `ops-finance` if it touches pricing, payments,
   or anything that needs terms/compliance.

Delegate each step to the right specialist rather than doing it all yourself.
End with what shipped, how to verify it, and the single most important next action.
